Design a premium commercial desktop web app called “Bridge”.

Bridge is an AI-powered understanding workspace that helps people make sense of documents, videos, audio, notes, links, images, and scattered context. The product should feel extremely simple to start, like opening a blank Figma file, but powerful as the user explores deeper.

Bridge should not feel like a chatbot, a dashboard, or an overwhelming AI tool. It should feel like a calm blank canvas where users can drop anything and Bridge quietly organizes, connects, explains, and simplifies the context for them.

The core product promise:
“Drop anything. Understand faster. Go deeper only when you want.”

Design the product as a commercial SaaS app with a polished onboarding, home/file system, blank workspace, canvas interaction model, and light/dark modes.

Visual direction:
Use a minimal, refined UI inspired by Figma, Wispr Flow, Gamma, Linear, and modern AI productivity tools.

The UI should feel:

* clean
* calm
* spacious
* commercial
* highly intuitive
* source-grounded
* human
* premium
* not AI-generated
* not visually noisy
* not overwhelming

Use this color palette:

* Warm off-white: #F6F7ED
* Lime accent: #DBE64C
* Deep navy: #001F3F
* Green: #00804C
* Soft green: #74C365
* Blue: #1E488F
* Charcoal for text and dark mode surfaces

Default to light mode first. Also create a premium dark mode using deep navy and charcoal surfaces, with lime and green accents used sparingly.

Typography:
Use clean modern typography. Keep text minimal. Use strong hierarchy, short labels, small chips, thumbnails, visual previews, icons, source cards, video/audio cards, and progressive disclosure instead of long explanations.

Core UX principle:
Bridge makes complex context feel simple. Do not show everything upfront. Start with almost nothing. Let the user add things freely. Bridge should organize itself in the background.

Avoid step-by-step wizard UI. Avoid forms unless absolutely necessary. Avoid asking too many questions at the start. The user should be able to begin by simply dropping a file, pasting a link, speaking, typing, or dragging media into the blank canvas.

“Drop files, links, notes, audio, or video. Bridge turns scattered context into clear understanding — with sources always attached.”

Primary CTA:
“Start a Bridge”

Secondary CTA:
“Watch quick demo”

Show a minimal animated product preview:
A blank canvas with a PDF, video, note, and image being dropped in. Thin connection lines appear softly. A small “Understanding ready” chip appears.

Do not overload this page. It should feel spacious and premium.

2. Getting Started screen
   Create a simple first-time experience.

Options as visual cards:

* Short summaries
* Source-first reading
* Audio walkthroughs
* Step-by-step checklists
* Deep research mode

Allow skipping:
“Skip — Bridge will adapt as I use it”

Then show:
“Create your first Bridge”
Options:

* Blank Bridge
* Upload files
* Paste a link
* Record or drop audio
* Start from a template

Keep this screen light, friendly, and fast.

4. Blank Bridge Canvas
   This is the most important screen.

When the user opens a new Bridge, show a mostly empty canvas.

The canvas should have:

* A minimal top toolbar
* A left source shelf
* A large blank working area
* A hidden right inspector that appears only when needed
* A small floating input dock at the bottom or center

Opening copy in the canvas:
“Drop anything here.”
Subcopy:
“Files, links, notes, images, videos, audio — Bridge will organize the context.”

Show only a few starting actions:

* Upload
* Paste link
* Write note
* Record
* Import

Make this feel as easy as adding a rectangle or text box in Figma. The user should not feel they are completing a setup flow.

5. Canvas interaction model
   Users can add anything directly to the canvas:

* PDFs
* Docs
* Slides
* Images
* Screenshots
* Audio
* Video
* Links
* Notes
* Voice notes
* Text snippets

Each input becomes a simple source card on the canvas.

Source card design:

* File thumbnail or preview
* Type icon
* Title
* Small processing state
* Tiny source count or duration
* Minimal actions on hover

Examples:

* PDF card with first-page thumbnail
* Video card with timeline strip
* Audio card with waveform
* Image card with visual preview
* Note card with short text
* Link card with website preview

Micro-interactions:

* Dragging a file shows a soft lime drop zone
* Dropping creates a smooth card animation
* Cards gently arrange themselves
* Related cards show faint connection lines
* Processing appears as small breathing dots, not a loud loader
* Hovering reveals “Open”, “Ask”, “Connect”, “Summarize”
* Clicking a card opens deeper tools without leaving the canvas

6. The “Bridge is thinking” state
   AI processing should happen quietly in the background.

Do not show a complex list of AI agents by default.
Instead show a small calm status chip:
“Reading sources…”
“Finding links…”
“Noticing gaps…”
“Building your map…”

If the user clicks this chip, show a small transparent process popover:

* Sources read
* Key ideas found
* Possible gaps
* Conflicts noticed
* Visual map ready

Make it feel trustworthy but not technical.

7. Left-to-right progressive depth
   Design the workspace so the user moves from simple to deeper understanding naturally.

Left side = simple source shelf.
Center = visual canvas and simple understanding.
Right side = deeper explanation, evidence, source proof, contradictions, and controls.

Left panel:
Simple source list only.
Keep it narrow and calm.
Show:

* All sources
* Source type
* Processing state
* Read / unread state
* Small thumbnails

Center canvas:
The main board.
Show cards, clusters, maps, highlights, summaries, visual references, and simple outputs.
This is the user’s primary space.

Right inspector:
Hidden by default.
Appears only when the user clicks a source, insight, question, or connection.
Shows deeper information:

* source proof
* exact page / timestamp
* explanation
* related references
* missing context
* confidence
* contradictions
* actions

This creates a depth model:
simple on left → working understanding in center → deeper explanation on right.

8. Understanding output should be visual-first, not text-first
   Bridge should not put long summaries in front of the user by default.

Default outputs should be:

* visual cards
* source thumbnails
* key idea chips
* timelines
* image/video references
* audio snippets
* relationship lines
* compact “must know” cards
* progress states
* source highlights

Text should be short and optional.

Create output modes:

* Visual Map
* Quick Brief
* Source View
* Timeline
* Audio Walkthrough
* Checklist
* Deep Dive

The user can switch modes from a small segmented control, not a large menu.

9. Personalization without overwhelm
   Bridge should adapt to the user’s preferred way of understanding.

Show a small “View style” control:

* Visual
* Brief
* Audio
* Source-first
* Deep

For a visual learner, the interface should emphasize:

* images
* reference cards
* video frames
* diagrams
* connection maps
* color-coded source links
* less body text

For a text-heavy user, it can show:

* structured summaries
* detailed explanations
* expandable notes

Do not make personalization feel creepy.
Use copy:
“Bridge adapts the view, not the truth.”

10. Source-grounded understanding
    Every insight must be traceable to the original source.

Each generated card should have tiny source chips:

* From PDF
* From video
* From note
* Inferred
* Needs confirmation

Clicking a source chip opens the right inspector with:

* exact source preview
* page number
* timestamp
* highlighted quote or frame
* related source cards
* “Open original” button

Show source highlighting:
When a user clicks an insight, subtly highlight the connected source card and draw a faint line to it.

11. Context gaps and questions
    Bridge should ask questions only when needed, and inside the workspace.

Do not create a long questionnaire.
Instead show small contextual prompt cards.

Examples:

* “This seems important. Should it be marked as must-know?”
* “This video says the timeline changed, but the PDF has the older date.”
* “Who is this Bridge for?”
* “Do you want a visual version of this?”
* “Should this note stay private?”
* “Do you want to add voice context?”

Each prompt card has quick actions:

* Yes
* No
* Add note
* Mark private
* Ask later
* Ignore

These should feel like helpful nudges, not required steps.

12. Main Bridge workspace states
    Generate multiple states of the same workspace:

A. Empty state
A beautiful blank canvas with simple drop-anything prompt.

B. Sources added state
Several source cards on the canvas, softly connected.

C. Processing state
Minimal processing chips and progress signals.

D. Understanding ready state
Bridge creates a simple visual cluster:

* Main idea
* Must know
* Key sources
* Open questions

E. Deep inspector state

F. Visual map state
Concept nodes connected to sources, images, videos, and questions.

G. Audio/video-first state
Canvas emphasizes waveform, video frames, timestamps, and visual summaries.
